# Atelier — Claude Instructions

## Project
Personal portfolio site for an artist/animator. Built with Angular 22 (standalone components, signals).

## User context
- New to TypeScript and Angular
- Learn by doing — explain concepts as we build.
- Frame-by-frame animations.

## Code comments
Always add beginner-friendly comments to Angular-specific code (components, signals, routing, directives, etc.) explaining what it does and why. Skip comments on plain HTML/CSS/JS that is self-explanatory.

## Git
Never commit unless explicitly asked to. Always wait for the user to say so.

## Obsidian learning vault
After implementing each new Angular concept or feature, write or update a note in the vault at:
`~/Obsidian/LearningAngular/`

Use wikilinks (`[[Note Name]]`) to connect related notes. Folder structure:
- `Concepts/` — core Angular ideas
- `Cookbook/` — how-to recipes tied to things we actually built
